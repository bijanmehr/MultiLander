Press P in the game and a neural network of 308 numbers flies the lander. This page explains every moving part — the network, the learning algorithm, the tricks, and the honest results. There is no black box here: the forward pass below is the actual shipped code, and it is short enough to read over coffee. The same pure-Python module that trained on a laptop runs live in your browser through Pyodide.

Sixty times a second, the same three beats as any pilot — sense, think, act: 

OBS(14 floats) ──► NETWORK ──► ARGMAX ──► {NOOP | LEFT | RIGHT | THRUST} ──► PHYSICS ──► repeat

Python builds a 14-number summary of the world, the network turns it into one of four button presses, the physics integrates one tick. The network has no memory, no planning, no lookahead — it is a pure reflex: the same 14 numbers always produce the same button.

The 14 inputs (full formulas in the manual): position, velocity, attitude (sin/cos + spin), fuel fraction, the vector to the nearest pad, pad half-width and multiplier, terrain clearance, and a pad-visible flag. All world-size-relative, most within ±1. Press O in the game — also while the AI flies — and you watch these exact numbers stream through its synthetic eyes. 

  obs[14] ──► [ 16 tanh units ] ──► [ 4 logits ] ──► argmax ──► action

  w1: 16×14 = 224      b1: 16      w2: 4×16 = 64      b2: 4
  total: 308 numbers — the whole pilot fits in a tweet-sized JSON file

The forward pass, verbatim from src/moonlander/core/policy.py (stdlib only — this exact code runs in your browser): 

# argmax index -> classic controls — EXACTLY env.py's Discrete(4) mapping:
# 0 noop, 1 rotate left (+1 = CCW), 2 rotate right (-1), 3 thrust.
ACTIONS = ((0, False), (1, False), (-1, False), (0, True))

def act(self, obs):
    """obs (14 floats) -> (rotate, thrust). Ties pick the lowest index."""
    h = [math.tanh(sum(w * x for w, x in zip(row, obs)) + b)
         for row, b in zip(self.w1, self.b1)]
    logits = [sum(w * v for w, v in zip(row, h)) + b
              for row, b in zip(self.w2, self.b2)]
    return ACTIONS[max(range(4), key=logits.__getitem__)]

That is the entire pilot. tanh squashes each hidden unit to (−1, 1); argmax picks the strongest of four opinions; ties go to the lowest index, so the flight is perfectly deterministic.

The cross-entropy method (CEM), the simplest learning algorithm that genuinely works here, in one breath: guess 64 networks, fly each one, keep the 10 best, aim your next guesses at the winners, repeat 60 times. No gradients, no backpropagation — just a bell curve over all 308 weights that slowly tightens around whatever flies well. The refit step, verbatim from src/moonlander/train_cem.py: 

def cem_update(samples, fitnesses, elite_k):
    """One CEM step: (mean, std) of the elite_k best samples (higher = better)."""
    order = np.argsort(fitnesses)[::-1]
    elite = samples[order[:elite_k]]
    return elite.mean(axis=0), elite.std(axis=0)

And the generation loop around it:

for gen in range(1, args.gens + 1):
    # Common random numbers: every candidate flies the SAME episode seeds
    # this generation — candidates compete on skill, not terrain luck.
    seeds = [int(s) for s in rng.integers(0, 2**31, size=args.episodes)]
    samples = rng.normal(mean, std, size=(args.pop, n))
    fitnesses = np.array([
        np.mean([episode_return(env, unflatten(th, args.hidden), s)
                 for s in seeds])
        for th in samples
    ])
    mean, std = cem_update(samples, fitnesses, args.elite)
    std += args.noise
    probe = np.mean([episode_return(env, unflatten(mean, args.hidden), s)
                     for s in probe_seeds])
    if probe > best_probe:
        best_probe, best_mean, best_gen = probe, mean.copy(), gen
  • Fitness is the raw episode return of the game's shaped reward — progress toward a pad, a small fuel cost, a big terminal bonus or penalty. Nothing else.
  • Common random numbers: all 64 candidates fly the same terrains each generation, so selection measures skill, not terrain luck.
  • Noise floor (std += 0.02): the bell curve is never allowed to collapse — it keeps exploring forever.
  • Checkpoint, not finish line: the wandering mean is probed each generation on held-out terrains, and the artifact ships the best report card, not the last one. (The first three training runs shipped the final generation and landed 1–2/30 — the dashed PROBE line below is why that was the wrong checkpoint.)

Gradient methods (REINFORCE, PPO) differentiate through the policy and need a discount factor γ to weigh future reward. This project's pre-training audit found a beautiful trap there: at γ = 0.99 a landing is ~1300+ decisions away, the terminal bonus discounts to ~0.0002, and hovering provably beats landing — a PPO agent with default settings would have converged to the hover, correctly optimizing the wrong objective.

CEM cannot fall into that trap, by construction: its fitness is the plain undiscounted sum of an episode's rewards. There is no γ anywhere in the trainer. The flip side is honest too: searching 308 weights by guessing scales terribly — at GPT scale this would be hopeless. That is exactly why this network is tiny, and why the roadmap's next phase (a PPO baseline) exists: gradients are what you graduate to when guessing stops being enough.
